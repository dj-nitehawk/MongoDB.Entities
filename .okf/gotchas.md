# Gotchas

- `DB.Default` is determined by the first initialized database for the default client settings. Changes to initialization order can change behavior.
- `DB.InitAsync` caches `MongoClient` instances by `MongoClientSettings`; be careful with equality/identity implications when changing client cache logic.
- Tests need replica-set MongoDB behavior for transactions and change streams. A standalone local MongoDB will not cover all features.
- `Tests/.mongo-keyfile` is generated local/CI material. Do not preserve key contents or treat it as source.
- Referenced relationships require saved parent/child entities before `Many.AddAsync`; unsaved entities do not have usable IDs.
- Relationship manipulation and file upload/download inside transactions require passing the transaction session to lower-level APIs where documented.
- Custom entity ID types are broadly supported by `IEntity`, but referenced relationships document support only for selected ID types.
- Partial entity save/upsert behavior can cause data loss if custom ID generation creates duplicates. Docs strongly recommend default `ObjectId` IDs.
- `Documentation/api/` and `Documentation/_site/` are generated by DocFX. Prefer source/XML/wiki edits plus regeneration.
- Library C# language version is intentionally `13`; do not bump casually because the project file says DocFX does not yet support C# 14.
- `SharpCompress` is present with a project comment saying it should be removed when MongoDB driver upgrades to a non-vulnerable version. Check the current dependency context before changing it.
- Avoid copying local test credential values from repo files into new documentation or OKF files.
